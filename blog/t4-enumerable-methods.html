Group_by and Cycle Methods

October 12, 2014

Just learned some great Enumerable methods. Enumerables are basically a collection of elements that you can use "each" methods on. You can click on the links to read more about Enumerable and Each methods. Enumerables are often Ranges and Arrays.

The first method I want to talk about is Group_by. What this method do is loop through a collection of elements, assign each element to a variable inside a block and group those elements in a Hash based on the result from the methods in the block.

(collection of elements).group_by{|variable| block} #=> a Hash where Keys are different results and Values are grouped elements who yield the same results.

Example 1. collection is an Array: [1,2,3,4,5,6]
[1,2,3,4,5,6].group_by{|number| number > 3}
Each number in the array will be assigned to the variable 'number', and then compared to see if it's greater than 3, if it is, that number will yield 'true' and all numbers that yield 'true' will be grouped together inside the retuned Hash while all the 'false' will be in a separate group. Please see result below
#=> {false=>[1,2,3], true=>[4,5,6]}

Example 2. collection is a Range: (1..20)
(1..20).group_by{|number| number%2==0 and number%4==0}
#=> {false=>[1,2,3,5,6,7,9,10,11,13,14,15,17,18,19], true=>[4,8,12,16,20]}, all numbers between 1 to 20 that are both divisible by 2 and 4 are grouped as 'true', while those that are not grouped as 'false'

Those other method that I want to talk about is Cycle. What this method does is loop through the collection of elements and performs methods inside the block n times. If n is not specified, then cycle will continue forever!

(collection of elements).cycle(n){|variable| block}

Example 1: collection is an Array: [1,2,3,4,5,6]
[1,2,3,4,5,6].cycle(2){|number| print number}. This will print '123456' twice consecutively. Ruby will return #=> '123456123456'. This is because we indicate to repeat he cycle 2 times on the array, and each element in the array will be printed. This don't seems very useful right?

Example 2: collection is an Array: ["Yes","NA","No"]
choice=["Yes","NA","No"].cycle. We assigned the array with infinit cycle to variable choice. It's now an instance of an Enumerator. We can use it by typing 'choice', and if we call 'choice.next' in Ruby, it'll return the first element "Yes", if we call 'next' on choice again, it'll return "NA", and again will return "No", and again it'll go back to "Yes". How will this be used?, I don't know but it's sure interesting.

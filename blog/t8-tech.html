SQL Injection

November 7, 2014

SQL is used to retrieve and update database and is used to display data on many web page. Web sites let users type in the search values to return relevant information from their database, like username and passwords. If you search YouTube for SQL injection, you will see a lot of video telling you what that is and see it in action. SQL injection is to add SQL command into a website’s user input fields and explore the structure of the web site’s database. Once understanding how a database is structured, like getting as list of table names and respective column names, one can retrieve wide range of information from the database, even to update or delete information from the database. Watch the video and see it in action.

There are ways to prevent SQL injection and one way is to have a list of words used in SQL command check against user input and disallow use of those words from the user. But SQL have very common words so this method is ineffective and inconvenient. A better way to protect a web site from SQL injection is to use SQL parameters on fields and user inputs that request information from relevant field will be check against its parameter. Here at W3school.com have some tips on how to use parameters for protection against SQL injection: W3School. Setting parameter is also known as Prepared Statements. OWASP also have cheat sheet to prevent SQL injection: Cheat Sheet

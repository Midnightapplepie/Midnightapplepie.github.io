Ruby classes vs Javascript constructor

November 2, 2014

Javascript don't have classes and constructor is used to create somthing like a class. Let's see how it's done in Javascript.

In Ruby:


			(to create a class)
			class Person
				attr_accessor :name, :age, :gender
				def initialize(name,age,gender)
					@name=name
					@age=age
					@gender=gender
				end
			end

			=====================================================
			(to create an instance of a class)
			tim=Person.new("Tim",22,"male")

			(to access instance variable of "tim")
			tim.name   #=>  "Tim"
			tim.age    #=>  22
			tim.gender #=> "male"

In Javascript, we'll use constructor:


			(you create a function named "Person", not a class)
			function Person(name, age, gender){
				this.name=name;
				this.age=age;
				this.gender=gender;				
			}

			=====================================================
			(check what is Person)
			typeof Person #=>'function'

			(to create a new object based on the Person function)
			var tim= new Person("Tim",22,"male")

			(to access properties of the object "tim")
			tim.name   #=>  "Tim"
			tim.age    #=>  22
			tim.gender #=> "male"

			(what is "tim"?)
			typeof tim #=>'object'

			("tim" is an instance of?)
			tim instanceof(Person) #=>true

Conclusion:
Person in our Javascript example is a function to create an object, these type of function are called constructors and the objects created under these function will be an instances of the function's name.
